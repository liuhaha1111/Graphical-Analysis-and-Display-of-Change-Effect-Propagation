# 产品设计知识图谱与变更传播主视图设计

## 背景

当前项目已经完成统一 `WorkspaceModel`、知识图谱派生服务和确定性传播分析服务，但两个核心页面仍停留在“列表化展示”阶段：

- 产品设计知识图谱页主要是节点列表与关系清单，还不是可操作的图谱主视图。
- 变更效应传播可视化页主要依赖路径卡片列表，尚未把知识图谱升级为传播分析的主承载界面。

根目录下的 `KnowledgeGraph.tsx` 与 `PropagationSimulation.tsx` 提供了旧版 canvas 原型参考，说明系统原本期望用图形化方式表达拓扑与传播。但现有 `src/features/*` 实现没有继承这部分价值。

本轮设计目标是把“知识图谱”从辅助信息区升级为两个页面的核心主视图，并保证传播分析仍然保持确定性、可解释、可测试。

## 目标

- 让产品设计知识图谱页成为统一拓扑探索界面。
- 让变更效应传播页以知识图谱为主视图承载传播结果。
- 保持产品域、参数域、供应链域和路由域在同一张图上可读。
- 用稳定分层布局替代随机自由布局，保证演示与测试稳定。
- 支持路径高亮、受影响节点过滤、节点详情联动和风险可视化。

## 非目标

- 不做复杂力导向布局。
- 不做实时拖拽编辑图谱。
- 不做超大规模图的性能优化。
- 不做路径结果的重新随机仿真。
- 不做图像/PDF 导出。

## 总体方案

采用“统一图谱数据 + 稳定分层布局 + 共享主画布 + 页面模式切换”的方案。

### 核心原则

1. 知识图谱是两个页面共享的主视觉对象，不再各自维护一套图逻辑。
2. 布局必须稳定、可解释，优先表达传播方向而不是追求网络散点效果。
3. 传播页只是在统一图谱上叠加分析结果，不重新定义一张新图。
4. 页面本地只保留 UI 状态，业务数据和传播结果继续由共享模型与服务层提供。

## 页面定位

### 产品设计知识图谱页

定位为“全局探索视图”：

- 左侧：筛选区，控制节点类型、关系类型、风险等级、组件阶段。
- 中间：知识图谱主视图，展示组件、参数依赖、供应伙伴、物流/路由节点。
- 右侧：节点详情、上下游关系、供应映射和约束摘要。

该页面回答的问题是：

- 这个产品和供应链网络整体长什么样？
- 某个组件或伙伴位于哪里？
- 某个节点与哪些对象存在显式关系？

### 变更效应传播可视化页

定位为“分析执行视图”：

- 左侧：场景面板，配置变更源组件、源参数、变更类型、变更幅度和显示过滤。
- 中间：传播知识图谱主视图，直接承载分析后的受影响节点、传播边和当前路径。
- 右侧：风险指标、传播路径列表、节点或路径详情。

该页面回答的问题是：

- 这次变更从哪里开始？
- 它沿着哪些链路传播？
- 当前最重要或风险最高的路径是什么？

## 主图布局

主图采用固定分层拓扑，而不是随机自由布局。

推荐层级从左到右为：

`变更源组件/参数 -> 受影响组件 -> 供应伙伴 -> 路由/物流节点`

在知识图谱页中，即使未执行传播分析，也沿用这一稳定分层，以保证认知一致性。在传播页中，布局不因每次分析而跳动，用户可以把注意力放在高亮变化上，而不是重新识图。

### 参数呈现策略

参数不默认平铺成独立主节点，否则图会迅速膨胀。

首版策略：

- 默认把参数作为组件节点的内嵌标签或简要摘要。
- 在需要强调传播来源或命中路径时，再按需展开为子节点或高亮标签。
- 传播路径中的参数阶段必须仍然可见，但以“局部展开”的方式出现。

## 视觉编码

### 节点编码

- 组件节点：圆角矩形，偏琥珀色，表示产品设计域。
- 供应伙伴节点：圆形或胶囊形，偏天蓝色，表示供应链域。
- 路由/物流节点：菱形，偏青绿色，表示运输或流转域。
- 变更源节点：固定高亮外环，作为视觉起点。

### 传播状态编码

- 未受影响：低饱和、较高透明度。
- 已受影响：使用 `浅橙 -> 橙 -> 红` 的热度渐变表达影响值。
- 当前选中路径节点：描边加粗、亮度提升、阴影增强。
- 非当前路径节点：保留在图中，但降噪显示，防止丢失全局上下文。

### 边编码

- `BOM`：细实线，灰橙色。
- `参数依赖`：虚线，金色。
- `组件 -> 供应伙伴`：蓝色实线。
- `供应链路由`：青色带箭头实线。

在传播页中：

- 边粗细映射影响强度。
- 当前路径边高亮。
- 其他边降低透明度但不彻底消失。

## 交互设计

### 通用交互

- 点击节点：右侧详情切换到该节点，显示属性、上下游和关联关系。
- Hover 节点：临时高亮一跳上下游关系。
- 支持缩放、平移、重置视图。
- 支持图例和过滤模式切换。

### 过滤模式

首版统一支持四种主过滤模式：

- 显示全部
- 仅受影响节点
- 仅当前路径
- 仅高风险节点

过滤只影响展示，不改写底层分析结果。

### 知识图谱页特有交互

- 默认显示全量网络。
- 允许筛选组件阶段、关系类型、伙伴风险等级。
- 选中节点后展示其供应映射与潜在风险入口，但不叠加传播热度。

### 传播页特有交互

- 执行分析后，主图默认进入“仅受影响节点 + 最高风险路径高亮”模式。
- 点击路径卡片：主图聚焦并强化对应路径。
- 点击节点：主图维持路径高亮，右侧切换到节点影响详情。
- 如果保留播放，仅做结果回放动画，不改变分析结论。

## 数据流设计

继续以 `WorkspaceModel` 作为唯一业务源。

### 服务层分工

- `knowledgeGraph.service.ts`
  负责把组件、参数依赖、伙伴和路由转换为统一图谱节点与边。
- `topologyLayout.service.ts`
  负责根据节点域、路径上下文和关系类型分配稳定坐标与层级。
- `propagationAnalysis.service.ts`
  继续负责确定性传播分析，输出路径、指标和命中阶段。

### 页面叠加模型

传播页不生成第二份图数据，而是在统一图谱之上叠加：

- 节点影响值
- 节点受影响状态
- 边命中状态
- 当前选中路径高亮状态
- 风险等级标签

## 组件拆分建议

### 共享图谱组件

- `src/components/graph/TopologyCanvas.tsx`
  渲染节点、边、状态高亮、缩放和平移。
- `src/components/graph/GraphLegend.tsx`
  渲染节点和边的图例说明。
- `src/components/graph/GraphToolbar.tsx`
  渲染过滤模式、显示开关和重置操作。

### 知识图谱页

- `src/features/knowledge-graph/KnowledgeGraphPage.tsx`
  作为页面编排容器。
- `src/features/knowledge-graph/KnowledgeGraphCanvas.tsx`
  退化为知识图谱模式的画布装配层，内部复用 `TopologyCanvas`。
- `src/features/knowledge-graph/KnowledgeGraphSidebar.tsx`
  负责筛选器与节点列表。

### 传播页

- `src/features/propagation-analysis/PropagationAnalysisPage.tsx`
  继续作为页面容器。
- `src/features/propagation-analysis/PropagationCanvas.tsx`
  改造成传播知识图谱主图容器，内部复用 `TopologyCanvas`。
- `src/features/propagation-analysis/ImpactSummary.tsx`
  保留指标摘要。
- `src/features/propagation-analysis/ImpactDetailList.tsx`
  改为辅助信息区，不再承担主视图职责。

## 状态划分

以下状态应保留在页面或图谱组件本地，而不进入 `WorkspaceModel`：

- `selectedNodeId`
- `selectedPathId`
- `graphFilterMode`
- `showParameterNodes`
- `affectedOnly`
- `viewport`（缩放与平移）

业务数据、变更场景和分析结果仍保留在共享 store。

## 错误处理与降级策略

### 未执行分析

- 传播页主图展示完整图谱，但不叠加热度。
- 右侧显示“等待执行分析”。
- 变更源节点保持预选高亮。

### 分析无结果

- 主图不空白。
- 仅高亮变更源节点。
- 右侧明确说明“未发现超过阈值的传播路径”。

### 数据缺失

- 缺失节点或边时跳过渲染，不抛运行时错误。
- 详情面板显示缺省文案。
- 可选地在摘要中给出缺失计数。

### 筛选后无内容

- 显示空状态遮罩。
- 保留图工具栏与“重置筛选”。

## 首版实现边界

### 首版必须实现

- 统一图谱主视图
- 稳定分层布局
- 传播热度叠加
- 路径高亮
- 节点详情联动
- 基础筛选、缩放、重置视图

### 首版不实现

- 自由拖拽改图
- 复杂自动避让
- 多路径并行动画
- 导出图片/PDF
- 大规模图虚拟化

## 验证标准

- 知识图谱页可以稳定展示完整拓扑并支持选择节点。
- 传播页执行分析后，主图能明确展示受影响节点与当前路径。
- 点击路径卡片、节点和过滤项时，主图和右侧详情保持同步。
- 无分析结果、无筛选结果、数据缺项时页面可用且无报错。

## 结论

本轮最合适的方案是：

- 用稳定分层拓扑替代自由散点图。
- 让知识图谱成为两个页面共享的主画布。
- 让传播页在统一图谱上叠加分析热度与路径高亮。

这样既能满足“这两部分都需要能展示知识图谱”的目标，也能避免再次把知识图谱页和传播页做成两套分裂的视图体系。
