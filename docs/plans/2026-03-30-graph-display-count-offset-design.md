# 图谱展示计数偏移设计

## 背景

当前项目中的知识图谱相关页面会直接把真实图数据的节点数和边数展示给用户：

- 主知识图谱页左侧概览展示全量实体数与 `可见关系 / 全量关系`
- 图谱画布工具条展示当前视图中的 `nodes / edges`
- 3D 场景在 WebGL 不可用时展示当前场景中的 `Nodes / Edges`
- 根目录旧版 `KnowledgeGraph.tsx` 原型页也展示实体总数与关系总数

本次需求要求：

- 在实体总数基础上额外展示 `+500`
- 在关系总数基础上额外展示 `+7000`
- 不能真的生成额外节点和边
- 其他页面里出现的同类展示也要统一变化

这意味着需求只影响“用户看到的展示计数”，不影响真实图结构、筛选、布局、交互或测试数据本身。

## 目标

- 统一将所有面向用户的图谱节点计数展示为 `真实值 + 500`
- 统一将所有面向用户的图谱边计数展示为 `真实值 + 7000`
- 保持真实图数据、筛选逻辑、布局逻辑和交互逻辑完全不变
- 避免在多个页面内重复手写偏移计算，保证展示口径一致

## 非目标

- 不修改 `buildKnowledgeGraphView` 生成的真实节点与边
- 不修改任何布局、密度判断、筛选、子图展开、节点回退逻辑
- 不引入新的假节点、假边或 UI 专用图数据副本
- 不改写服务层关于真实图规模的断言

## 方案对比

### 方案一：每个页面单独手写偏移

在各组件里把 `nodes.length`、`edges.length` 直接改成 `+500 / +7000`。

优点：

- 修改快

缺点：

- 容易漏掉入口
- 相同规则分散在多个文件中，后续维护差
- 不同页面可能出现不同口径

### 方案二：增加统一展示计数工具

新增一个专门的展示层工具，把真实节点数和边数转换成展示值，再由各 UI 组件调用。

优点：

- 偏移规则集中管理
- 不影响底层图数据和业务逻辑
- 方便统一处理单值展示与 `visible / total` 双值展示

缺点：

- 需要遍历所有展示入口并替换

### 方案三：构造带偏移计数的包装图对象

在页面层创建一个额外对象，把偏移计数挂在图对象上供各处读取。

优点：

- 个别页面接入代码可能更少

缺点：

- 容易被误用到布局、密度判断或筛选逻辑中
- 混淆真实数据和展示数据的边界

## 结论

采用方案二：新增统一展示计数工具，只在 UI 展示层应用偏移。

## 详细设计

### 展示计数工具

新增 `src/services/graphDisplayMetrics.ts`，只提供纯函数：

- `displayEntityCount(count)`：返回 `count + 500`
- `displayRelationCount(count)`：返回 `count + 7000`
- `displayEntityCountLabel(count)`：返回适合直接展示的实体数字符串
- `displayRelationCountPair(visible, total)`：返回偏移后的 `visible / total` 字符串

该工具只负责展示口径，不依赖 React，不读取工作区状态，不修改图对象。

### 适用范围

以下用户可见计数入口统一走展示计数工具：

- `src/features/knowledge-graph/KnowledgeGraphSidebar.tsx`
  - 实体总数
  - 关系总数
  - 关系类型筛选列表中的 `visible / total`
- `src/components/graph/GraphToolbar.tsx`
  - 当前视图的 `nodes`
  - 当前视图的 `edges`
- `src/components/graph/TopologyScene3D.tsx`
  - WebGL fallback 的 `Nodes / Edges`
- 根目录 `KnowledgeGraph.tsx`
  - 实体总数
  - 关系总数

### 保持不变的逻辑

以下逻辑继续使用真实图数据，不允许改用展示偏移值：

- `buildKnowledgeGraphView`
- `filterKnowledgeGraphByRelationTypes`
- `buildKnowledgeGraphSubgraph`
- `buildTopologyLayout`
- `buildTopology3DLayout`
- 画布密度判断
- 空状态判断
- 展开一跳
- 当前焦点与当前选中节点回退逻辑

## 展示口径

### 实体计数

- 全量实体数：`真实节点总数 + 500`
- 当前视图节点数：`真实当前节点数 + 500`

### 关系计数

- 全量关系数：`真实边总数 + 7000`
- 当前可见关系数：`真实可见边数 + 7000`
- 关系筛选明细中的各类型 `visible / total` 也都使用上述口径

## 边界约定

- 即使当前可见边数为 `0`，展示值也应为 `7000`
- 即使当前局部子图节点很少，工具条仍展示 `真实当前节点数 + 500`
- 空状态文案不受影响，仍由真实图数据决定是否为空
- 服务层与布局层测试继续断言真实值，不断言展示偏移值

## 风险与控制

### 风险一：展示逻辑渗透到底层图逻辑

控制方式：

- 把偏移函数放在独立展示工具里
- 不在服务层类型中增加偏移字段
- 不修改任何图派生或布局服务

### 风险二：不同页面展示口径不一致

控制方式：

- 所有页面统一调用同一个展示工具
- 为工具函数与主知识图谱页补测试

## 验收标准

- 用户在所有相关图谱页面看到的节点计数均为真实值基础上 `+500`
- 用户在所有相关图谱页面看到的关系计数均为真实值基础上 `+7000`
- 实际节点数仍为 `3012`
- 实际边数仍为 `5573`
- 关系筛选、布局、展开、空状态行为与当前一致
