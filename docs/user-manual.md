# 操作手册

## 1. 文档说明

本文档用于指导当前项目的启动、页面操作和结果查看。手册内容严格基于当前代码实现编写，重点描述现有可用操作，不包含尚未实现的业务流程。

## 2. 运行准备

### 2.1 环境要求

- Node.js
- npm

### 2.2 安装依赖

在项目根目录执行：

```bash
npm install
```

### 2.3 启动开发环境

在项目根目录执行：

```bash
npm run dev
```

启动后，在浏览器访问：

```text
http://localhost:3000
```

### 2.4 常用命令

```bash
npm test
npm run build
npm run lint
```

说明：

- `npm test`：启动 Vitest 测试
- `npm run build`：执行生产构建
- `npm run lint`：当前项目用于 TypeScript 类型检查

## 3. 界面总览

系统首页主要分为两个层级：

1. 顶部页签区
   用于切换三大模块：
   - 产品结构建模
   - 产品设计知识图谱
   - 变更效应传播可视化
2. 下方内容区
   根据当前页签展示对应模块页面

三个模块共享同一份工作区数据，因此页面之间看到的组件、供应伙伴和变更源是关联的。

## 4. 通用操作

### 4.1 切换模块

操作步骤：

1. 打开系统首页
2. 点击顶部任一模块卡片
3. 页面切换到对应功能区

### 4.2 查看卡片信息

每个模块页面由多张卡片构成。卡片顶部通常包含：

- 模块名称
- 简要说明
- 统计指标或模式信息

建议先查看页面顶部说明，再进入具体操作。

### 4.3 选择当前焦点对象

当前系统主要通过“点击节点”或“点击 BOM 项”切换焦点对象：

- 在产品结构建模中，点击 BOM 树中的组件
- 在知识图谱中，点击中部画布中的节点
- 在传播分析中，点击路径导航项或画布节点

## 5. 产品结构建模模块操作

### 5.1 功能目标

用于查看产品 BOM 层级、组件参数和参数依赖关系，为后续图谱浏览和传播分析提供基础背景。

### 5.2 页面组成

- 顶部统计区
  - 组件总数
  - 参数总数
  - 依赖链路数
- 左侧产品结构树
- 中间组件参数面板
- 右侧参数依赖关系面板

### 5.3 操作步骤

#### 步骤 1：选择组件

在“产品结构树”中点击任一组件卡片。

执行结果：

- 当前组件高亮
- 中间参数面板更新为该组件的数据
- 右侧依赖关系面板更新为该组件相关链路

#### 步骤 2：查看组件参数

在“组件参数”面板中查看：

- 组件类别
- 阶段
- 参数数量
- 传入依赖数量
- 传出依赖数量
- 参数基线值
- 约束范围
- 是否可变更
- 参数备注

#### 步骤 3：查看参数依赖关系

在“参数依赖关系”面板中查看：

- 源组件和源参数
- 目标组件和目标参数
- 关系类型
- 表达式
- 影响权重

同时可查看页面底部“变更约束摘要”，了解：

- 可调参数数
- 边界参数数
- 传入链路数
- 传出链路数

### 5.4 使用建议

- 优先选择带参数的组件进行查看，便于后续在传播分析模块中使用
- 若某个组件没有参数或依赖链路，页面会显示空状态提示，这是当前数据定义的正常表现

### 5.5 当前限制

当前页面仅支持查看，不支持新增、编辑、删除或保存产品模型。

## 6. 产品设计知识图谱模块操作

### 6.1 功能目标

用于从统一拓扑角度查看产品组件、供应伙伴和物流路线之间的关系。

### 6.2 页面组成

- 左侧图谱概览
- 中部拓扑画布
- 右侧节点详情

### 6.3 图例说明

当前图中主要对象如下：

- 产品结构节点
- 供应链节点
- 关系边

关系边在系统中主要对应：

- BOM 关系
- 供货映射关系
- 物流路径关系

### 6.4 操作步骤

#### 步骤 1：查看网络概览

在左侧“图谱概览”中查看：

- 节点总数
- 产品节点数
- 供应链节点数
- 关系总数
- 供货映射数
- 物流路径数

#### 步骤 2：点击节点

在中部“拓扑画布”中点击任一节点。

执行结果：

- 选中节点高亮
- 左侧“当前焦点”更新
- 右侧“节点详情”同步更新

#### 步骤 3：查看节点详情

若选中的是产品组件，可查看：

- 组件说明
- 类别
- 阶段
- 标签
- 参数数量

若选中的是供应伙伴，可查看：

- 位置
- 角色
- 风险等级
- 能力侧重点

所有节点都可以查看：

- 与当前节点相连的边
- 连接对象名称
- 连接类型

### 6.5 使用建议

- 当画布内容较多时，可通过容器滚动查看全部节点
- 当前图谱布局是系统自动生成的固定排布，不支持手动拖拽调整

### 6.6 当前限制

当前页面不支持：

- 节点拖拽
- 缩放
- 关系编辑
- 导出图谱图片或数据

## 7. 变更效应传播可视化模块操作

### 7.1 功能目标

用于配置变更场景并执行一次确定性传播分析，查看变更对产品域和供应链域的影响。

### 7.2 页面组成

- 左侧变更场景注入
- 中部传播画布
- 右侧影响摘要与影响明细

### 7.3 操作步骤

#### 步骤 1：选择源组件

在“变更场景注入”中的“源组件”下拉框选择一个组件。

执行结果：

- 当前分析源组件更新
- 若该组件存在参数，“源参数”下拉框会同步刷新
- 中部画布会将该节点作为当前焦点之一

注意：

- 某些组件当前没有参数定义
- 如果切换后“源参数”为空，建议重新选择带参数的组件

#### 步骤 2：选择源参数

在“源参数”下拉框中选择需要作为传播起点的参数。

建议优先选择以下类型参数：

- 性能参数
- 容量参数
- 亮度/功耗相关参数

#### 步骤 3：设置变更类型

在“变更类型”中选择一种类型：

- 规格变更
- 成本变更
- 工期变更
- 质量变更

不同类型会影响系统内部的评分修正系数。

#### 步骤 4：设置变更幅度

在“变更幅度”中选择：

- 低
- 中
- 高
- 关键

幅度越高，通常影响基值越大。

#### 步骤 5：填写场景说明

在“场景说明”文本框中输入本次变更背景。

该字段当前主要用于记录说明文本，不会单独形成持久化记录。

#### 步骤 6：执行分析

点击“执行传播分析”按钮。

执行结果：

- 系统生成传播路径
- 中部画布高亮受影响节点
- 右侧影响摘要显示风险指标
- 右侧影响明细显示路径导航和结论信息

### 7.4 结果查看方法

#### 影响摘要

分析完成后，可查看：

- 风险等级
- 受影响节点数
- 综合影响分
- 成本风险
- 工期风险

#### 路径导航

在“影响明细”中的“路径导航”区域点击任一路径。

执行结果：

- 当前路径高亮
- 中部画布突出显示该路径涉及的边
- “当前路径”区域展示阶段标签

#### 高亮结论

“高亮结论”区域用于快速浏览本次分析中的关键结论，例如：

- 变更源信息
- 影响到的组件和伙伴数量
- 风险级别

#### 结果明细

“结果明细”区域用于查看简要明细，包括：

- 成本风险和工期风险概览
- 受影响组件列表

### 7.5 使用建议

- 建议先在“产品结构建模”模块确认组件和参数，再进入分析页面
- 建议多切换“变更类型”和“变更幅度”对比结果变化
- 建议结合“路径导航”和“传播画布”一起解读影响链路

### 7.6 当前限制

当前页面不支持：

- 保存多个历史场景
- 导出分析报告
- 对比两次分析结果
- 自定义分析规则
- 接入实时业务数据

## 8. 常见问题

### 8.1 启动后页面无法访问

请检查：

- 是否已执行 `npm install`
- 是否使用 `npm run dev` 启动项目
- 是否访问了 `http://localhost:3000`

### 8.2 选择组件后没有源参数

原因：

- 当前示例数据中，部分组件没有参数定义

处理方式：

- 重新选择带参数的组件

### 8.3 刷新页面后分析结果丢失

原因：

- 当前系统使用内存态数据，不做持久化保存

处理方式：

- 重新配置场景并执行分析

### 8.4 为什么看不到导入、保存、导出功能

原因：

- 当前项目是演示型原型，暂未实现这些功能

## 9. 当前使用结论

当前系统适合用于：

- 原型演示
- 方案汇报
- 功能讨论
- 后续系统设计参考

当前系统不适合直接作为正式业务平台上线使用。
