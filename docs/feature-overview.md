# 功能说明

## 1. 文档目的

本文档基于当前项目代码编写，用于说明“变更效应传播图形化分析展示”原型系统的功能范围、模块职责、数据结构和当前版本边界。

## 2. 系统概述

本项目是一个基于 React + Vite 的前端演示原型，围绕同一份内存态 `WorkspaceModel` 展示三类能力：

1. 产品结构建模
2. 产品设计知识图谱
3. 变更效应传播可视化

系统当前使用内置示例数据驱动，不依赖后端接口，也不提供持久化存储。页面刷新后，场景设置和分析结果会恢复为默认状态。

## 3. 功能结构

### 3.1 顶层导航

系统首页顶部提供 3 个页签：

- 产品结构建模
- 产品设计知识图谱
- 变更效应传播可视化

用户通过页签在三个模块之间切换。三个模块共享同一份工作区数据，因此前两个模块中的结构和节点，会直接影响第三个模块的分析输入与图形展示。

### 3.2 产品结构建模

该模块用于浏览产品域数据，当前版本以“查看”和“选择”为主，不提供在线编辑能力。

当前页面包含以下区域：

- 概览卡片
  显示组件总数、参数总数、依赖链路总数。
- 产品结构树
  以 BOM 层级展示系统、总成、模块、零件之间的父子关系。
- 组件参数面板
  展示当前选中组件的参数基线值、上下限、是否可变更、备注说明，以及依赖数量统计。
- 参数依赖关系面板
  展示当前组件相关的参数传播链路，包括源组件/参数、目标组件/参数、关系类型、计算表达式和影响权重。

当前模块可支持的操作：

- 点击 BOM 树节点切换当前组件
- 查看组件阶段、类别、描述和标签
- 查看参数边界、变更性和备注
- 查看参数上下游依赖和约束摘要

当前模块不支持：

- 新增、删除、编辑组件
- 新增、删除、编辑参数
- 导入 BOM、Excel、CSV
- 保存建模结果

### 3.3 产品设计知识图谱

该模块将产品组件、供应伙伴和物流路径融合为统一的拓扑视图，用于观察设计域和供应链域之间的映射关系。

当前页面包含以下区域：

- 图谱概览
  显示节点总数、关系总数，以及当前焦点节点名称。
- 拓扑画布
  以统一画布展示产品节点、供应链节点和三类边：
  - BOM 边
  - 供货映射边
  - 物流路径边
- 节点详情
  展示当前选中节点的详细信息。
  - 若为产品组件节点，显示类别、阶段、标签、参数数量
  - 若为供应链节点，显示位置、角色、风险等级、能力侧重点
  - 同时展示该节点的连接关系

当前模块可支持的操作：

- 点击图谱节点查看详情
- 观察产品节点与供应链节点之间的连接关系
- 查看当前网络中的节点数量与边数量
- 查看图例说明

当前模块不支持：

- 节点拖拽排布
- 缩放、框选、多选
- 图谱编辑和关系修改
- 图谱导出

### 3.4 变更效应传播可视化

该模块是当前原型中交互最完整的部分，用于配置变更场景并执行确定性传播分析。

当前页面包含以下区域：

- 变更场景注入
  用于设置分析输入参数。
- 传播画布
  在统一拓扑图上高亮受影响节点和当前路径对应的边。
- 影响摘要
  展示风险等级、受影响节点数、综合影响分、成本风险和工期风险。
- 影响明细
  展示路径导航、高亮结论、结果明细和当前路径阶段。

当前模块可配置的场景字段：

- 源组件
- 源参数
- 变更类型
  - 规格变更
  - 成本变更
  - 工期变更
  - 质量变更
- 变更幅度
  - 低
  - 中
  - 高
  - 关键
- 场景说明

当前模块可支持的操作：

- 选择变更源组件和参数
- 修改变更类型和变更幅度
- 输入场景说明
- 执行传播分析
- 在路径导航中切换不同传播路径
- 在画布中点击节点查看当前焦点

分析输出内容包括：

- 受影响节点总数
- 传播路径集合
- 综合影响分
- 成本风险
- 工期风险
- 风险等级
- 高亮结论
- 结果明细

## 4. 分析逻辑说明

当前分析逻辑为“确定性规则分析”，不是机器学习模型，也不是基于实时业务数据的仿真系统。

分析过程大致分为三段：

1. 从源参数出发，沿参数依赖链路向其他组件参数传播
2. 将组件影响映射到供应伙伴
3. 沿供应链物流路径继续传播

分析计算会综合以下因素：

- 变更幅度对应的基础系数
- 变更类型对应的影响、成本、工期修正系数
- 参数关系类型对应的权重
- 参数链路自身的影响权重
- 供应商风险等级
- 供货提前期
- 物流路线可靠性
- 物流运输时长

最终输出“低 / 中 / 高 / 极高”四档风险等级。

## 5. 共享数据模型

系统三大模块共享同一份 `WorkspaceModel`，包括以下三部分：

### 5.1 产品域

- `components`
  产品组件清单，包含层级关系、类别、阶段、说明等信息
- `parameters`
  组件参数清单，包含基线值、约束范围、单位、是否可变更
- `parameterLinks`
  参数依赖链路，描述参数间传播方向、关系类型、表达式和影响权重

### 5.2 供应链域

- `partners`
  供应伙伴清单，包含角色、位置、风险等级、供货能力等信息
- `routes`
  供应链物流路径，包含运输方式、时长、可靠性和约束

### 5.3 分析域

- `changeScenario`
  当前变更场景
- `analysis`
  当前分析结果

## 6. 当前示例数据范围

当前内置示例以一台笔记本产品为基础，数据范围包括：

- 9 个产品组件
- 7 个参数
- 3 条参数依赖链路
- 5 个供应链伙伴
- 3 条物流路线

默认分析场景为：针对 CPU 频率提升的规格变更场景。

## 7. 当前版本边界

### 7.1 已实现

- 三模块统一工作区模型
- BOM 浏览与组件详情查看
- 产品与供应链统一图谱展示
- 确定性传播分析
- 路径高亮与影响摘要展示

### 7.2 未实现

- 用户登录与权限控制
- 后端接口
- 数据库存储
- Excel / CSV 导入导出
- 在线编辑和保存模型
- 多用户协同
- 报告导出
- 历史场景管理

## 8. 适用场景

当前项目更适合以下用途：

- 演示“共享数据模型驱动多个视图”的产品原型
- 演示产品域与供应链域联动的可视化思路
- 演示变更影响传播分析的基础方法
- 用作后续业务系统设计、汇报或二次开发的原型基础

不适合作为正式生产系统直接投入业务使用。
